Guia Rápido de Git & GitHub

Como usar o controlo de versões no nosso projeto Astro Pi com o VS Code.

1. O Conceito (Git vs. GitHub)

É simples: o Git é o programa no teu PC que tira "fotografias" (snapshots) do teu código. O GitHub é o site (a "nuvem") onde guardas essas fotografias online para partilhar com a tua equipa e ter um backup.

  • Repositório (Repo): A pasta do teu projeto.
  • Commit: Uma "fotografia" ou um "ponto de gravação" (save point) do teu código.
  • Push: Enviar os teus commits locais para o GitHub.
  • Clone: Copiar um repositório do GitHub para o teu PC.

2. Configuração Inicial (Só fazes 1 vez)

Antes do teu primeiro commit, tens de dizer ao Git quem tu és. Abre o terminal (no VS Code: Terminal > New Terminal) e corre estes dois comandos: 


# Define o teu nome de utilizador (usa o teu nome real ou nick)
git config --global user.name "O Teu Nome"

# Define o teu email (o mesmo que usaste no GitHub)
git config --global user.email "o.teu.email@exemplo.com"

3. O Workflow da Sessão (O Ciclo)

Em cada sessão, vamos seguir este ciclo. 99% do tempo, vais usar os botões do VS Code, que fazem estes comandos por ti.

Clone (Só na Sessão 1) → Programar (Editar Código) → Stage (Add) → Commit → Push

4. Comandos Essenciais (Terminal vs. VS Code)

Podes usar os comandos no terminal ou os botões no painel "Source Control" (⛉) do VS Code.

git clone

Copia um repositório do GitHub para o teu PC. (Só fazes isto na Sessão 1). 


# Copia o link do botão verde "Code" no GitHub
git clone https://github.com/a-tua-equipa/astro-pi.git

No VS Code: Vais fazer isto manualmente no terminal na primeira vez.

git add

"Prepara" os ficheiros que alteraste para a próxima "fotografia" (commit). Diz ao Git "quero incluir esta alteração". 


# Adiciona um ficheiro específico
git add main.py

# Adiciona todos os ficheiros alterados (usa com cuidado!)
git add .

No VS Code: Clicar no ícone **"+"** ao lado do ficheiro no painel "Source Control". (Chama-se "Stage Changes").

git commit

Tira a "fotografia" (grava) dos ficheiros que preparaste. Exige sempre uma mensagem a descrever O QUE mudou. 


git commit -m "S1: Estrutura inicial do main.py"

No VS Code: Escrever a mensagem na caixa de texto do painel "Source Control" e clicar no **"✓" (Commit)**.

git push

Envia os teus commits (as tuas "fotografias" guardadas) do teu PC para o GitHub, para a tua equipa ver. 


git push

No VS Code: Clicar no botão **"Sync Changes"** ou "Push" na barra azul no fundo do ecrã.

git status

O comando mais útil. Mostra-te *exatamente* que ficheiros modificaste e o que não está guardado (commitado). 


git status

No VS Code: O painel "Source Control" (⛉) é um `git status` visual e em tempo real.
